☁ Cloud Subscriptions Help

What are Cloud Subscriptions?

Cloud Subscriptions displays live Azure cloud assets that have been harvested from your environment using the Azure CLI. This provides a real-time view of your provisioned resources, their public exposure status, and architectural relationships.

🚀 Getting Started

To populate the Cloud Subscriptions page, you need to run the harvest scripts: 

# Login to Azure
az login

# Harvest subscriptions and assets
python Scripts/Harvest/harvest_azure_assets.py --subscription "My-Prod-Sub"

# (Optional) Correlate assets with your repositories
python Scripts/Harvest/correlate_assets.py --subscription "My-Prod-Sub"

Note: Replace My-Prod-Sub with your actual Azure subscription name.

📋 Harvest All Subscriptions

To harvest all accessible subscriptions at once: 

python Scripts/Harvest/harvest_azure_assets.py --all

🏛️ Understanding the Table

  • Subscription — Azure subscription name and ID
  • Provider — Cloud provider (e.g., Azure)
  • Environment — Environment type (prod, dev, test, etc.)
  • Total Assets — Number of provisioned resources
  • Public — Number of publicly exposed resources (⚠ indicates risk)
  • State — Subscription state (Enabled, Disabled, etc.)
  • Last Synced — When assets were last harvested

📐 View Architecture

Click the View Architecture button on any subscription to see:

  • Resource Groups and their contents
  • Ingress routing and entry points
  • Network topology and connectivity
  • Interactive zoom, pan, and export controls for analysis

⚙️ Prerequisites

  • Azure CLI: az command must be installed
  • Authentication: Run az login before harvesting
  • Permissions: Your Azure account must have read access to subscriptions and resources
