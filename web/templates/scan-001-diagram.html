Key Components

Virtual Network

Core network boundary containing all internal resources and traffic routing.

Network Security Group

Network access control rules and security policies for traffic filtering.

Virtual Machine

Compute resource with publicly-exposed IP for application hosting.

Storage Account

Blob storage for application data and static content.

CosmosDB

NoSQL database for application state and data persistence.

Function App

Serverless compute for business logic and event-driven operations.

⚠️ Security Considerations

Public VM Access — Virtual machine exposed to internet via public IP
Storage Public Access — Storage blob containers with public access enabled
CosmosDB Public Endpoint — Database accessible from internet
Overly Permissive NSG Rules — Network security group allows broad traffic patterns
Function App Exposed — Automation functions accessible without authentication
